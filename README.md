# aya-assistant
Браузерный ассистент, созданный по подобию Claude for Chrome.

Использует модель claude-haiku-4-5-20251001

## Установка:
- Открыть управление расширениями в Chrome
- Включить режим разработчика (тумблер в правом верхнем углу)
- Загрузить неупакованное расширение (Load unpacked), указав папку 'extension'
- Активировать расширение как любое другое

## Использование:
- Нажать кнопку для ввода API ключа в правой части хедера
- Ввести ключ и сохранить его
- Отправить задачу через поле для ввода снизу

## Архитектура:
Довольно стандартный вариант для агентов

User input -> Claude Haiku -> Runtime execution -> Claude Haiku

Модель отправляет команды в рантайм -> Выполнение команд -> Получение результата выполнения моделью

## Функционал:
- Выполнение как простых, так и многошаговых задач
- Механизм рассуждений и автоматического исправления собственных ошибок
- Удобный чат и работа внутри штатного браузера
- Работа на запросах к Anthropic API
- Использование вводимого пользователем собственного API токена

## Чего не хватает:
- Security layer для подтверждения действий со стороны пользователя
- Task states для оптимизации траты токенов, краткая запись проделанных и оставшихся шагов, состояния страницы
- Запрос информации от пользователя
- Защита от спрятанных инструкций, перезаписывающих текущие

В данной первоначальной версии возможны баги и непредвиденные ситуации. Модель полностью рассуждает самостоятельно и может делать ошибки.

### Комментарий от разработчика
К сожалению, Claude for Chrome на данный момент не работает и я не могу имплементировать функционал 1 в 1 и даже сравнить работу моего и ассистента Anthropic.

Была проведена максимальная оптимизация самих промптов, много экспериментов с разными подходами. На данный момент выбран наиболее эффективный из опробованных, но он все еще требует достаточно много токенов для решения задачи. Считаю, что для данной версии продукта он работает очень неплохо.

Почему я выбрал JS и DOM запросы? Данное решение было принято в связи с невозможностью запускать Chrome (не chromium!) с заранее подгруженным расширением, легким обнаружением автоматизации при использовании playwright (капчи правда очень мешали) и удобством использования (терминал vs браузерное расширение).

Основные баги касаются скорее UI части расширения, так как ее почти полностью сгенерировал ИИ, с минорными правками своими руками.
Рекомендую использовать данное расширение только на доверенных сайтах, так как единственная защита от спрятанных инструкций в коде страницы - это вшитая в модель антропиков.
